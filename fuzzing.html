Overview

The TP Protocol Fuzzing Suite provides comprehensive fuzz testing capabilities for critical network protocols including RTP, RTCP, RTSP, and TLS. Designed for robustness, it executes up to 100,000 randomized cases per run, ensuring thorough coverage through bit-bounded sampling and sequential field iteration.

Supported Protocols

üì°

RTP

Real-time Transport Protocol

  • Distribution: Bit-bounded sampling (e.g., 32-bit timestamp ~40k).
  • Targeting: Sequential only.
  • Connectivity Loss: Connectionless; diagnose via device logs or Wireshark.
  • Packet Recording: Logs contain full packet content required to reproduce frames.

üì∂

RTCP

RTP Control Protocol

  • Distribution/Targeting: Same as RTP.
  • Uniqueness: Full-run evidence in log.txt.
  • Connectivity Loss: Connectionless; use device logs / Wireshark.
  • Packet Recording: Same rationale as RTP.

üé¨

RTSP

Real Time Streaming Protocol

  • Distribution: Cases allocated by command; each run shuffles/adds/removes/mutates commands.
  • Targeting: Per-command isolation not supported; tested in order.
  • Connectivity Loss: Connection-oriented; stops on drop. Use timestamps to locate triggering packet.
  • Recording: Logs focus on command payloads.

üîí

TLS

Transport Layer Security

  • Distribution: 50k fixed-header + 50k fully randomized.
  • Targeting: No direct jump to specific field/strategy.
  • Connectivity Loss: TLS over TCP‚Äîany disconnection interrupts transmission.
  • Logging/Wireshark: Fixed-header phase (encrypted payloads) vs Full-fuzz phase (visible payload bytes).
