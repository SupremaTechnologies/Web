Overview

AspNetWebShield is a high-performance RASP (Runtime Application Self-Protection) solution engineered specifically for ASP.NET environments. Unlike traditional WAFs (Web Application Firewalls) that operate at the network perimeter, AspNetWebShield runs inside your application. By injecting directly into the .NET runtime environment using advanced DLL hooking technology, it provides real-time monitoring and blocking of deep-level attacks including WebShell injections, Remote Code Execution (RCE), Memory Shells, and Sensitive File Tampering.

Zero-Day RCE Defense

Deep hooks into low-level APIs like Process.Start and ShellExecute to stop attacks before they execute.

  • Command Blocking: Automatically identifies and blocks the execution of malicious system commands such as cmd.exe, whoami, calc, and other reconnaissance tools.
  • Context Awareness: Analyzes execution parameters and the full Stack Trace in real-time to distinguish between legitimate application behavior and RCE exploits.

Advanced WebShell & Memory Shell Protection

Built-in heuristics designed to counter popular WebShell management tools (e.g., AntSword, Godzilla, Behinder).

  • Memory Assembly Scanning: Intercepts Assembly.Load(byte[]) to prevent "Fileless" Memory Shells from loading malicious payloads directly into RAM.
  • Reflection Monitoring: Hooks MethodBase.Invoke to detect reflection-based attacks. It specifically flags known malicious class/method signatures (e.g., BASE_Info, CMD_Exec, ExploitSuggest).
  • Instant Block: Terminates the request immediately upon detecting attack signatures and logs the attempt.

Real-Time File Integrity Monitoring (FIM)

Integrated active file surveillance for your web root and codegen directories.

  • Change Detection: Instantly triggers a file_warning alert upon the creation or modification of high-risk file types like .aspx and .dll.
  • Tamper Proofing: Hooks File.Copy, File.Delete, and Directory.CreateDirectory to prevent attackers from modifying system files or wiping their tracks.

Memory Integrity Verification

Equipped with self-defense mechanisms to ensure the security module itself hasn't been bypassed or tampered with by a Rootkit.

  • NTDLL Hash Check: Periodically calculates the SHA1 hash of the ntdll.dll .text section in memory and compares it against the disk image to detect unhooking attempts.
  • Anomaly Alerting: Sends immediate alerts if core system libraries are modified in memory.

Sensitive Operation & Anti-Spyware

  • Anti-Screenshot: Intercepts Graphics.CopyFromScreen to prevent malware from capturing server screen contents.
  • Script Blocking: Monitors Microsoft.JScript.Eval to prevent unauthorized server-side script execution.

Technical Specifications & Architecture

  • Core Engine: Built upon the robust HarmonyLib and MonoMod frameworks for stable, high-performance .NET method hooking.
  • Auditing & Logging:
    • Local Logs: Categorized logging into log.txt (General), warning.txt (Threats), ex.txt (Exceptions), and file_warning.txt (FIM).
    • Remote Syslog: Supports UDP forwarding (Port 514) to integrate with SIEM/SOC solutions (Default IP: 192.168.0.101).
  • Licensing System:
    • RSA Encryption: Uses asymmetric encryption for secure license validation.
    • Hardware Binding: Binds the license to the server's CPU ProcessorId to prevent unauthorized copying.
    • Expiration Control: Built-in validity period checks to manage subscription lifecycles.

Deployment & Operations

  • Drop-in Installation: Simply deploy the DLLs to your application's bin directory.
  • Configuration: Manage settings via config.json to define license paths and log output directories.
  • Auto-Protection: The system automatically validates the license, initializes the Hook Engine, and starts File Watcher threads upon application startup.
  • Active Blocking: When an attack is intercepted, the attacker receives a "Blocked" response page displaying the specific attack type and stack trace, serving as a powerful deterrent.
